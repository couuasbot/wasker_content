# 项目概览
TRICYS（TRitium Integrated CYcle Simulation）是面向聚变堆氚燃料循环的模块化、多尺度仿真平台，强调基于物理的闭环分析与全系统质量守恒，帮助研究人员评估氚管理方案并优化系统设计。

## 架构设计
### 1. 物理建模核心
- **OpenModelica + Modelica**：使用 Modelica 建模，借助 OpenModelica 进行求解。
- **动态模型库**：覆盖等离子体排气处理、同位素分离与燃料供给等关键子系统。

### 2. 仿真编排与参数扫描
- **Python 驱动编排**：通过 CLI 统一调度场景运行与批量试验。
- **多参数扫描**：支持并发批量计算，快速探索参数敏感性。

### 3. 协同仿真与外部工具桥接
- **子模块协同**：提供与外部工具（如 Aspen Plus）的数据交换接口。

### 4. 分析与报告层
- **自动化报告**：生成包含图表、统计与结论的 Markdown 报告。
- **敏感性与 AI 增强**：结合 SALib 与 LLM，将原始数据转化为结构化分析。

### 5. 开发与交付环境
- **Windows 优先**：满足协同仿真与工业软件集成需求。
- **Docker Dev Container**：提供标准化 0D 仿真与可复现开发环境。

## 源码与架构深度解析
### 逻辑源码模块
- **Modelica 模型库**：沉淀可复用子系统组件（等离子体排气、同位素分离、燃料供给等），可组合成完整燃料循环模型。
- **Python 编排层**：定义场景配置、校验参数范围，并通过 `tricys` CLI 统一调度执行。
- **数据管线与持久化**：将仿真输出规范化为结构化数据，便于统计、对比与敏感性分析。
- **分析与报告工具链**：汇总指标、生成图表与表格，并输出 Markdown 报告供复核。
- **外部工具适配**：面向协同仿真工具（如 Aspen Plus）提供边界条件与结果交换接口。

### 端到端数据流
1. **场景定义**：组合 Modelica 组件并打包参数集，形成可复现实验场景。
2. **执行与采样**：编排层运行单次或参数扫描任务，采集时序输出。
3. **标准化处理**：对输出数据进行统一结构化与归一化，支撑后续分析。
4. **洞察生成**：报告聚合关键指标、假设与趋势，并结合 AI 总结结论。
5. **生态联动**：结果可推送至 [[tricys-backend]] 服务层，再进入 [[tricys-visual]] 可视化端。

## 工作流
1. **环境准备**：安装 Python、OpenModelica 与依赖库。
2. **场景配置**：组合 Modelica 组件并设置参数。
3. **仿真执行**：通过 `tricys` CLI 运行单次或批量扫描任务。
4. **结果分析**：自动生成报告与敏感性分析结论。
5. **迭代优化**：基于输出反馈调整模型与策略。

## 生态协同
- 与 [[tricys-backend]] 联动，完成任务调度与数据服务。
- 向 [[tricys-visual]] 提供仿真结果与可视化数据。
